# 블로그 작성 가이드

## 블로그 스택

- Astro + AstroPaper 템플릿
- 포스트 위치: `src/data/blog/`
- 언어: 한국어
- 배포: Cloudflare Pages (https://kim-bong-seop.pages.dev)

## 참고 문서 → 블로그 글 변환 규칙

참고 문서를 기반으로 블로그 글을 쓸 때 **원문의 사실과 논리 구조를 그대로 유지**한다.

### 절대 바꾸면 안 되는 것

- **사실 관계** — 원문에 있는 판단, 결론, 수치, 비교를 다르게 서술하지 않는다
- **맥락 추론 금지** — 원문 주변 맥락에서 없는 내용을 추론해 삽입하지 않는다. "A와 B를 같이 썼다"는 문장에서 "A가 B를 호출했다"를 추론하는 것은 금지
- **저자의 입장** — 원문에 없는 과거 실수, 오판, 시행착오, 구조적 전환을 추론해서 추가하지 않는다
- **서사 생성 금지** — 원문이 단순 비교 판단("A보다 B가 적합하다")일 때, "A를 쓰다가 B로 바꿨다"는 before/after 서사를 만들지 않는다

### 허용되는 변환

- 문서 형식(목차, 표, 섹션 구조)을 블로그에 맞게 재구성
- 딱딱한 문어체를 구어체로 전환
- 내부 참조 링크(Part 2, Part 3 등)를 간략화하거나 제거
- 독자 이해를 돕는 흐름 연결 문장 추가 (단, 원문의 사실을 벗어나지 않을 것)

### 확인 절차

변환 후 원문과 대조해 다음을 검토한다:
1. 저자가 내린 핵심 판단이 동일하게 표현되었는가
2. 원문에 없는 내용이 추가되었는가
3. 원문의 비교·대안 탐색 결과가 뒤집혀 있지 않은가

## 마크다운 작성 주의사항

- `~` 기호는 GFM에서 취소선 마커로 해석될 수 있으므로 본문에서 직접 사용하지 않는다
  - 범위 표현 (`A~B`) → `A부터 B`
  - 대략의 의미 (`~100줄`) → `약 100줄`
  - 경로(`~/.claude/`)는 반드시 백틱으로 감싼다: `` `~/.claude/` ``
